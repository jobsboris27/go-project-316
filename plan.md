## Задача 1: Возможность задавать скорость обхода

### Что нужно сделать

| Подзадача | Статус | Примечание |
|-----------|--------|------------|
| 1.1. Документировать CLI-параметры `--delay` и `--rps` | ⏳ pending | В README и коде |
| 1.2. Реализовать приоритет RPS над delay | ⏳ pending | Если оба указаны — использовать RPS |
| 1.3. Глобальное ограничение скорости (не на поток) | ⏳ pending | Один rate limiter на все запросы |
| 1.4. Тест на интервалы между запросами | ⏳ pending | Проверка, что интервалы ≥ заданного значения |
| 1.5. Отмена контекста не вызывает зависания | ⏳ pending | Проверка cancellation |

### Анализ текущего состояния

✅ **Уже реализовано:**
- Пакет `internal/shared/rate_limiter.go` существует

⚠️ **Проблемы:**
- Rate limiter создаётся один на воркер, но должен быть **общий на все запросы**

### План реализации

```

       - Проверить, что RPS имеет приоритет над delay

Шаг 2: Написать тест на rate limiting
       - Создать mock-сервер с подсчётом запросов
       - Запустить краулер с --rps=2 (500ms между запросами)
       - Замерить время между запросами
       - Убедиться, что интервалы ≥ 500ms

Шаг 3: Документировать в README
       - Добавить описание --delay и --rps
       - Указать приоритет RPS
```
